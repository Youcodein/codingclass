Scss

Sass(SCSS)는 Syntactically Awesome StyleSheet의 약어이며 코드의 재활용성을 올리고, 가독성을 올리는 등 CSS에서 보이던 단점을 보완하고, 개발의 효율을 올리기 위해 등장한 CSS 전처리기 언어입니다.

사용 이유

작업이 크고 고도화 될수록 CSS를 사용할 때 불편함이 생깁니다. 이에 SCSS와 Sass는 CSS 작업을 쉽게 해주며 가독성과 재사용성을 높여주어 유지보수가 쉬워지게 도와줍니다.

또한 Sass는

변수의 사용
조건문과 반복문
Import
Nesting
Mixin
Extend/Inheritance

다음과 같은 추가 기능과 유용한 도구들을 제공합니다.

CSS와 비교하였을 때 Scss는
1. CSS보다 심플한 표기법으로 CSS를 구조화하여 표현 가능.
2. 스킬 레벨이 다른 팀원들과의 작업 시 발생할 수 있는 구문의 수준 차이를 평준화 가능.
3. CSS에는 존재하지 않는 Mixin 등의 강력한 기능을 활용하여 CSS 유지보수 편의성을 큰 폭으로 향상 가능.

이러한 장점들이 있습니다.

사용 방법

웹에서는 CSS만 인식하기 때문에 Sass와 같은 전처리기 언어는 직접 동작되지 않습니다. 우선 전처리기 언어 문법으로 작성(코딩)하고, CSS로 컴파일해서 웹으로 동작시켜야 합니다.
